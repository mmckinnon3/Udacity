Stage 1 Notes

-Html is the structure of a web page
-A web page is basically a series of boxes filled with conent
-<div> defines a section in an HTML doc
-HTML and coding in general is done in a tree like structure
    -This means that it is branching.
    -You can include elements within elements, nesting.
    -An example of nesting would be
    <div>
        <p>This is a paragraph.</p>
    </div>
    - The above is an example of one element in another, but it is not limited to that
    -You can nest multiple elements within each other.

CSS Styling

-CSS is styling for html
-Using CSS you can change the color, font, or text size
-The CSS file should be in the same folder as the HTML file
    -You point the HTML file to the CSS file using the <link> element.
    -doesn't have to be, but you have to link the full path
-A class is a selector included in an HTML element. example: <div class="content">
    -Within the CSS file you define stylistic properties to classes, headers, or other elements in the html document.
    -for example, the code below included in a css file would color the text within all elements containing the CSS selector "content" blue.
        .content{
                    color: blue;
        }
-It is possible to stlye text and all other attributes within the HTML document without using a css file at all.
-Styling using CSS is preffered because it reduces repetition when creating code.
-Reducing repetition is important for multiple reasons
    -CSS styling allows you to quickly make a style change to many items in one html page with minimal coding. For instance, I can change all headers to green from blue by simply changing the word from green to blue one time in the css file.
    -If I were to style each element in the html doc, I would have to change that style numerous times, creating additional work for myself.
    -People make mistakes, by using repetition you reduce the amount of code you have to write, thereby reducing the opportunity to make mistakes.

Box Sizing

4 main points
-HTML elements are boxes and each box has 4 components.
-Because there are so many components to each box, it can often be hard to get the size of a box just right.
-There are two techniques you can use to help deal with sizing issues.
    -Set sizes in terms of percentages rather than pixels.
    -Set the box-sizing attribute to border-box for every element.
-Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.

Box Positioning

-Divs are block elements (as opposed to inline), so by default they take up the entire width of a page.
-Adding the rule display: flex; to the appropriate CSS will override this behavior and let divs appear next to each other.
